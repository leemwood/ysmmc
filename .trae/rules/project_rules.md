# 项目规则

## 通用规则
- 当用户提出新的要求或者获得新的资源知识时，将内容保存至本项目目录下的 `.trae/rules/project_rules.md` 文件中。
- 必须保持 `project_rules.md` 文件更新。
- 不要使用 emoji 表情符号。
- 始终使用简体中文。

## 开发规范
- 前端框架：Vue.js 3 + TypeScript
- 构建工具：Vite
- 样式方案：SCSS，统一管理样式文件
- 数据库/后端：Supabase
- 后端框架：Go + Gin + GORM + PostgreSQL

## 安全规范

### 环境变量
- 必须设置 `DB_PASSWORD` 和 `JWT_SECRET` 环境变量
- `JWT_SECRET` 长度必须至少 32 个字符
- 使用 `.env` 文件管理配置，不要硬编码敏感信息

### CORS 配置
- 通过 `ALLOWED_ORIGINS` 环境变量配置允许的域名（逗号分隔）
- 生产环境禁止使用 `*` 通配符

### 认证授权
- 使用 JWT 进行身份验证
- Access Token 默认有效期 24 小时
- Refresh Token 默认有效期 7 天，存储在数据库中
- 支持主动吊销 Refresh Token

### 文件上传
- 模型文件限制为 `.ysm` 和 `.zip` 格式
- 图片文件限制为 `.jpg`, `.jpeg`, `.png`, `.gif`, `.webp` 格式
- 验证文件魔数（Magic Number）确保文件类型正确
- 文件名长度限制 255 字符
- 使用 UUID 作为存储文件名

### 速率限制
- 登录接口：每 IP 每分钟最多 5 次
- 注册接口：每 IP 每小时最多 3 次
- 忘记密码：每 IP 每小时最多 3 次

### 输入验证
- 用户名：2-50 字符，仅支持字母、数字、下划线、连字符和中文
- 密码：至少 6 个字符
- 邮箱：标准邮箱格式验证
- 所有用户输入需进行 XSS 过滤

## 存储规范

### 目录结构
```
uploads/
├── models/          # 模型文件
│   └── 2026-03/    # 按日期分区（可选）
├── images/          # 图片文件
│   └── 2026-03/    # 按日期分区（可选）
└── temp/            # 临时文件
```

### 配置项
- `UPLOAD_PATH`: 上传根目录（默认 `./uploads`）
- `MAX_FILE_SIZE`: 最大文件大小（默认 100MB）
- `MAX_DISK_USAGE`: 最大磁盘使用率百分比（默认 90%）
- `ENABLE_DATE_PARTITION`: 是否按日期分目录（默认 false）

### 存储服务
- 服务启动时自动创建上传目录
- 上传前检查磁盘空间
- 支持按日期分目录存储
- 支持临时文件清理（超过 24 小时）

## 权限控制

### 角色验证
- `isValidRole()` 白名单检查 `user` 和 `admin` 角色
- 普通管理员不能设置 `admin` 角色
- 禁止修改超级管理员角色
- 禁止通过 API 分配 `super_admin` 角色
